# RimR UI 风格规范

RimR 使用 Web 技术栈（React + Tailwind CSS）构建桌面工具，但视觉与交互上应尽可能贴近原生桌面应用，避免典型的 Web 默认风格。

## 1. 圆角策略

- **全局 `--radius` 保持 `0`**，默认不提供任何圆角。
- 只允许在两类元素上使用极轻微的圆角（`2px`，通过 `rounded-slight`）：
  - **按钮（`Button`）**
  - **section 边界**：卡片（`Card`）、弹窗（`Dialog`）、侧边面板（`Sheet`）、顶部菜单下拉、列表/面板容器等。
- 输入框、文本域、徽章、菜单项、列表项、滚动条等保持直角。
- 不允许 pill（全圆角）徽章或标签。

## 2. 紧凑密度

- 列表行高度保持紧凑（如 active/inactive mod 行高 `h-7`，group child `h-6`）。
- 避免无意义的 `space-y-4`、`gap-4`、`p-4`；主视图优先使用 `gap-2`、`p-2`、`space-y-3`。
- 卡片内边距克制（`py-4 px-4`），头部与内容之间用细边框而非大留白分隔。
- 信息密度优先于“呼吸感”。

## 3. 按钮组合

- 一排功能相关的连续按钮应当合并成按钮组，消除它们之间的 gap 和重复边框。
- 使用 `ButtonGroup` 包裹连续按钮；组内只有第一个和最后一个按钮保留轻微圆角，中间按钮完全直角并共享边框。
- 典型场景：工具栏的 Reset/Save/Apply、排序 select + 方向按钮、卡片底部的 Reset/Save。

## 4. 层级与操作入口

### 主视图保持干净

- 主视图只放状态、切换、全局动作。
- 不放用于编辑的文本输入框、大段诊断文本，或大量操作按钮。
- 禁止控制台式布局：不把日志、完整诊断、配置表单直接铺在主视图。

### 善用二级界面

- **ContextMenu**：列表行的主要编辑入口（添加、移除、分组、重命名、删除等）。
- **顶部菜单**：跨模组列表/库设置的全局动作（导入、导出、视图切换、主题、语言）。
- **Dialog**：所有需要文本输入或确认破坏性操作的场景。
- **Sheet / 侧边面板**：展示只读详情（如 Catalog 的 ModDetails）。
- **Tooltip / title**：图标按钮必须带有 `title` 或 tooltip 说明，避免歧义。

### 避免平铺按钮

- 同一区域出现 3 个以上相关动作时，优先考虑菜单或下拉菜单。
- 例如 Catalog 的 source filter 使用单个下拉菜单，而非 4 个平铺按钮。

## 5. 颜色与阴影

- 优先使用 `border` 和 `background` 区分区域，减少大面积阴影。
- 按钮可保留细微投影以区分层级，但避免模糊的 Web 式大阴影。
- 保留焦点 `ring` 以支持键盘导航，但不使用圆角装饰。

## 6. 字体与排版

- 正文使用系统默认无衬线字体，保持可读。
- 技术字段（packageId、sourceKey、路径等）使用等宽字体 `font-mono`，字号可略小。
- 列表中的辅助信息使用 `text-muted-foreground`，避免视觉喧宾夺主。

## 7. 组件使用约定

- `Button`：使用 `rounded-slight`；图标按钮必须带 `title`/`aria-label`。
- `ButtonGroup`：合并连续按钮，消除 gap 与重复边框。
- `Badge`：不使用圆角，保持直角标签。
- `Card` / `Dialog` / `Sheet`：section 边界使用 `rounded-slight`。
- `Input` / `Textarea`：直角边框，无阴影。
- `DropdownMenu` / `ContextMenu`：菜单容器可带轻微圆角，菜单项直角。
- `ScrollArea`：滚动条 thumb 方形。

## 8. 反模式

- 不要在列表项、徽章、输入框上使用圆角。
- 不要在列表项之间使用大圆角卡片或阴影卡片。
- 不要在主视图直接放置文本编辑表单。
- 不要把诊断/日志以控制台样式堆叠在主视图。
- 不要为每个动作都提供一个可见按钮；优先上下文菜单或按钮组。
